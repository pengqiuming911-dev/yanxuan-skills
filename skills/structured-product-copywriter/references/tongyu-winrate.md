# 通毓终端取胜率流程

本文档说明如何通过通毓量化终端(`terminal.tongyu-quant.com`)的"结构化产品回测"页,自动获取一个结构化产品的**回测胜率**,用于填补文案里的胜率数据。胜率是文案标题/副标题的核心卖点,必须真实可复现,绝不能编。

## 何时用这个流程

仅当本地有 Playwright 浏览器控制能力(如 Claude Code 的 Playwright MCP)时才走本流程。没有浏览器自动化能力时,胜率仍用 `[胜率待补]` 占位,让用户手动提供——不要因为没法自动取就编一个数字。

## 凭证

**不要把账号密码写进技能文件或任何会被打包的文件里。** 凭证从以下来源读(按优先级):

1. 环境变量 `TONGYU_USER` / `TONGYU_PASS`
2. 本地文件 `~/.claude/tongyu-creds.json`(在技能目录之外,不会被 `.skill` 包打包),格式:
   ```json
   {"username": "...", "password": "..."}
   ```

都没有就问用户要,或退回 `[胜率待补]`。

## 防风控:登录前必读(必做,不是可选)

通余终端对自动化登录有滑块风控。触发一次,该会话基本废了,且短时间内再登会更易再触。下面每条都是**必做**,不是"尽量":

1. **凭证绝不手敲**。用户名和密码都用脚本从 `~/.claude/tongyu-creds.json` 或环境变量一次性读出,直接传入填值。绝不凭记忆/占位串手填——填错一次就是一次 422,422 紧接着重试是触发滑块的**头号原因**(实测:手敲错密码 → 422 → 改对重试 → 滑块立即弹出)。读凭证一行命令即可:
   ```bash
   python -c "import json,os; d=json.load(open(os.path.expanduser('~/.claude/tongyu-creds.json'),encoding='utf-8')); print(d['username']); print(d['password'])"
   ```
   读出的两行原样贴进填值,不要中间转述。
2. **用持久化浏览器 profile,复用登录态**。能复用就别每次全新登录——只有首次登录(或 cookie 过期后)才需要走登录流程,后续多次跑都直接带着 cookie 进站,**不再触碰风控**。有 wmux 浏览器能力的,用 `browser_session_start` 起一个命名 profile(如 `tongyu`),登录态在该 profile 里持久化;只有 Playwright MCP、没有 profile 管理的,登录成功后用 `storageState` 存到 `~/.claude/tongyu-session.json`,下次加载该 state 再进站。

   **纯 Playwright MCP 下的现实限制**:`browser_run_code_unsafe` 的沙箱拿不到 Node 的 `require`/`fs`,`storageState()` 取出来的 JSON 没法写进磁盘文件——所以"存到 `~/.claude/tongyu-session.json`"这条路在纯 MCP 下**走不通**,每次新会话都得重新登录。好消息是:只要照本节的防风控做(凭证脚本读、逐字输入、单次提交),登录通常很顺、不触滑块,重登代价不大。若想真正持久化,得用独立 Python 脚本(`playwright` 库 + `storage_state` 参数)而非 MCP——一般不值得为省这一次登录专门搭。
3. **逐字输入,不要瞬间 fill**。用户名/密码框用逐字键盘输入(Playwright `pressSequentially`;MCP 工具里 Playwright 的 `browser_type` 传 `slowly:true`;wmux 的 `browser_type` 传 `humanlike:true`)。不要用瞬间 `.fill()`——后者被风控识别为自动化的概率明显更高。
4. **单次干净提交**。点一次"登录账号/Login"就等 3-5 秒看结果,不要短时间反复点。

## 登录

1. 优先用持久化 profile 打开(见防风控第 2 条),已登录态会自动跳过登录直接到主页。
2. 需要登录时打开 `https://terminal.tongyu-quant.com/#/login`。顶部"管理员登录"按钮会跳到管理员门户(`/userManagement/`),**不是**我们要的入口——别点它。用页面中央的账号密码框。
3. **用上面那行命令从凭证文件读出用户名和密码**,逐字输入填入(防风控第 1、3 条),不要手敲、不要瞬间 fill。
4. 点"登录账号/Login"**一次**,等 3-5 秒。
5. 成功后 URL 跳到 `/#/`(主页)。**422 且响应体是 `用户名或密码错误`**:凭证错——**不要重试**,先 `cat` 凭证文件核对内容,改对后再单次提交。连续重试会触发风控。
6. 登录成功后,**立即保存登录态**(持久化 profile 自动存;若用普通 context,存 `storageState` 到 `~/.claude/tongyu-session.json`),下次免登录。**纯 Playwright MCP 下存不了**(见防风控第 2 条的说明),跳过这步直接进回测页即可。
7. **弹出滑块验证码("请完成下列验证后继续")**:见下一节,不要程序绕、也不要立刻放弃。

## 遇到滑块验证码:人工接管,不要放弃

滑块是反自动化风控,**绝不程序绕**。但也不必立刻退回占位——浏览器窗口是可见的,人工拖一下滑块就能救回这一跑:

1. **不要关闭浏览器**。在同一页面会话里告诉用户:"通余终端弹出滑块验证码,请在浏览器窗口里手动拖动滑块完成验证,完成后回我一声。"
2. 用户完成验证后,再次点"登录账号/Login"(用户名密码仍填着),通常即登录成功。
3. 登录成功后**立即保存登录态**(持久化 profile 自动存 / `storageState`),避免下次又走登录又触滑块。
4. **只有当用户不在场、无法人工拖滑块时**,才退回 `[胜率待补]` 占位,让用户日后自己手动跑一次把胜率告诉你。

为什么改为人工接管而非直接放弃:滑块弹出意味着这一跑已付出成本,人工 5 秒就能救回来;直接退回占位等于白跑。加上持久化 profile,这次登录态存下后,后续多次跑都不再触发——把"概率性失败"变成"一次性代价"。

## 服务器跑通方案:Xvfb 有头(绕滑块,实测可行)

无 GUI 服务器 headless 登录通徐必触滑块,但用 **Xvfb 虚拟显示器跑"有头"浏览器**不触滑块(实测:登录成功跳波动率页,无验证码;和本地有头一致)。openclaw 服务器已预装 Xvfb,且 `openclaw.json` 的 `mcp.servers.playwright` 已配 `xvfb-run` 包裹。

**服务器跑通徐胜率**(一行命令,全自动):
```bash
xvfb-run -a python3 scripts/tongyu_winrate.py --term 36 --ko 101 --step-down 0.5 \
  --parachute 60 --coupon-line 78 --coupon 1.39 --output assets/tongyu-winrate.png --headed
```
`--headed` 让浏览器有头(在 Xvfb 虚拟屏),登录不触滑块,填表+取胜率+截图全自动。实测胜率 98.14%,和本地一致。`xvfb-run -a` 自动分配虚拟显示器。

**注意**:`--headed` 在 Xvfb 下无真实屏幕,若极端触滑块(罕见)救不回→退 [胜率待补];但实测多次未触。AMAC/产品卡 headless 无风控,直接 headless 或同样用 xvfb-run 均可。

**前情**:headless(无 Xvfb)登录必触滑块(stealth/rm profile/wait 均无效),且登录态在 sessionStorage 不持久——所以 headless 不可行,必须 Xvfb 有头。

## 进入回测页

登录后导航到 `https://terminal.tongyu-quant.com/#/investmentAnalysis/InvestmentAnalysis`(标题"结构化产品回测")。左侧菜单"结构化产品回测"。

**导航重定向坑**:登录成功后(URL 跳到 `/#/`)直接导航到回测页 URL,可能被重定向到首页/波动率曲面页(投资分析 tab 未 active)。解法:先点顶部 tablist 的"投资分析"tab 让它变 `[active]`,再导航到回测页 URL,即可进入(标题变"结构化产品回测")。

若页面正中弹出"快捷查询"对话框(输入标的代码那个),先点右上角 close 关掉——它会遮挡表单点击。

## 填表:产品参数 → 表单字段映射

这是最关键的一步。表单字段会随"结构类型"选择动态变化,所以**先选结构,再填数**。映射表(左侧是用户给的参数,右侧是表单字段):

| 产品参数 | 表单字段 | 说明 |
|---|---|---|
| 结构类型 | 结构类型 → 组合结构类型 | 基础结构单选(DCN/经典/凤凰/早利/蝶变),叠加条款多选(降敲结构/降落伞结构/欧式结构等)。按产品勾选 |
| 标的 | 挂钩标的 | 搜索框输代码或名称;代码可用 `scripts/fetch_quote.py` 的 NAME2CODE 映射(中证1000=000852.SH, 沪深300=000300.SH, 中证500=000905.SH, 创业板指=399006.SH 等)。勾选完一项后不要再展开下拉框罗列其余选项 |
| 期限 | 期限(月) | 如 36 |
| 锁定期 | 有无锁定期 → 有锁定期,再填"锁定期(月)" | 如 3。选"有锁定期"后会多出锁定期月数字段。**无锁口径**:产品期限写"36M无锁"时,锁定期按 **1 个月**处理(选"有锁定期"填 1,或终端"无锁定期"选项,以字段语义"锁定期=1月"为准) |
| 期初敲出线 | 首次观察敲出价(%) | 如 101。即首月敲出线。**选了降敲结构后才会出现此字段** |
| 降敲 | 敲出价递减步长(%) | 如 0.5(每月)。即每月降敲比例。**选了降敲结构后才会出现** |
| 降落伞 | 期末障碍价(%) | 如 60。即敲入价/期末障碍价。**选了降落伞结构后才会出现**(DCN 下叫"期末障碍价") |
| 派息线 | 派息障碍价(%) | 如 78。DCN 下出现 |
| 费后派息 | 每月或有派息(%) | 如 1.39。DCN 下出现 |
| 是否追保 | 是否追保 → 不追保/追保 | 单选,一般选不追保 |
| 保证金 | 保证金水平(%) | 见下方"保证金坑" |
| 回测区间 | 回测区间 开始/结束日期 | 终端默认区间**随当天滚动**（实测 2026-07-11 跑出 2016-07-09~2026-07-08），不要套旧默认 2016-06-26~2026-06-25。脚本会打印 `date_range`，文案那句回测时间必须用这个实际值 |
| — | 价格区间 / PE区间 | 默认"全部",不动 |
| — | 开仓频率 | 默认"每日(交易日)",不动 |
| — | 敲出观察频率 | 默认"每月",DCN 下通常 disabled。按结构要素判断,一般每月 |

**基础结构选择规则**(按结构类型名映射,叠加条款默认勾欧式结构,结构里含降敲/降落伞再分别勾上):

- 结构名含 `DCN` → 基础结构选 DCN。
- 结构名含 `早利`(早利锁盈) → 基础结构选早利。
- 结构名含 `经典`(经典锁盈) → 基础结构选经典。
- 结构名含 `凤凰` → 选凤凰;`蝶变` → 选蝶变。
- 叠加条款:默认勾选欧式结构;结构要素里有降敲 → 勾降敲结构;有降落伞 → 勾降落伞结构。详见下方「欧式结构锁定坑」。

**通毓表单的「末次观察敲出价」字段**：选降敲后出现此字段，表单默认按"首次观察敲出价 − 步长 × 观察间隔数"自动联动成 85(末次敲出线)。但本产品里**末次观察敲出价 = 期末障碍价 = 降落伞**(如 60)，三者一致——脚本在填完「期末障碍价」后紧接着把「末次观察敲出价」也填成降落伞值(`args.parachute`)，覆盖默认的 85。若该结构无此字段(`set_by_label` 返回 `NOTFOUND`)忽略即可；若返回 85 说明没覆盖成功，检查字段定位。

**填数技巧**:antd 的 InputNumber 用 `.fill()` 通常能生效;若某个值没写进去,改用点击字段 → Ctrl+A 全选 → 逐字输入(pressSequentially)。填完扫一眼快照确认所有值。

## 保证金坑

选"不追保"后,保证金水平字段可能被**锁定**为站点默认值(如 40%),不可改。这是站点逻辑,不是 bug。**胜率由敲出/敲入结构(标的、期限、敲出价、敲入价、降敲、锁定期、回测区间)决定,保证金水平基本不影响胜率**——因为胜率算的是"合约能不能敲出/不敲入拿票息",与保证金杠杆无关。所以即便站点把保证金锁成 40%、产品实际是 50%,胜率数字仍可信;文案里保证金/杠杆仍按产品实际值写。

## 欧式结构锁定坑(DCN+降敲+降落伞组合下,取消不掉但可忽略)

选了 DCN + 降敲结构 + 降落伞结构 后,"叠加条款"里的**欧式结构会被强制锁定为 active**,无法取消:点击卡片(span.spantxt 或 `.snowball-structure` 父级)、点 check-circle 图标、键盘 Enter/Space、DOM `.click()`、Playwright 真实 click——均无效,span 始终带 `active` class。初始状态(经典+欧式)下点欧式也取消不掉。本产品(月度观察敲出)按理不该勾欧式,但取消不了。

**但不影响胜率可信度,接受现状即可**:实测选了 DCN+降敲+降落伞+欧式(锁定)后,"平均敲出时间"约 249 天(远小于 36 月期限),证明敲出仍是**月度观察**(非"只在期末观察"的欧式)。推测此处"欧式结构"指敲入观察方式(期末敲入),与降落伞(期末障碍价)一致,不改变月度敲出节奏。故**不要纠结取消欧式,直接跑分析**,胜率可信。

**判断欧式是否影响结果**:看"平均敲出时间",若远小于期限(如 249 天 vs 36 月),说明月度敲出生效,结果可信;若接近期限,才是真欧式,要再处理。

## 批量填值最稳方法(回测页 antd InputNumber)

回测页的 antd InputNumber 对 Playwright `fill` 生效(Enter 触发自动分析,**不弹 form submit alert**,和小工具不同)。批量填值用 `browser_run_code_unsafe` + `page.evaluateHandle` 按 label 文本定位 input,再 `click({clickCount:3})` 全选 + `fill(value)` + `press('Enter')`。模板(已验证期限/首次观察敲出价/期末障碍价/末次观察敲出价/派息障碍价/敲出价递减步长/每月或有派息 全部生效):

```js
async (page) => {
  const fields = [
    {label:'期限', value:'36'},
    {label:'首次观察敲出价', value:'101'},
    {label:'期末障碍价', value:'60'},        // 降落伞
    {label:'末次观察敲出价', value:'60'},    // = 期末障碍价 = 降落伞，覆盖表单默认自动联动的 85
    {label:'派息障碍价', value:'78'},         // 派息线
    {label:'敲出价递减步长', value:'0.5'},    // 降敲
    {label:'每月或有派息', value:'1.39'},     // 费后派息
  ];
  const out = {};
  for(const f of fields){
    const inp = await page.evaluateHandle((label)=>{
      let el=null;
      for(const e of Array.from(document.querySelectorAll('div,span,label'))){
        if(e.textContent.trim()===label){el=e;break;}
      }
      if(!el) return null;
      let c=el;
      for(let i=0;i<6;i++){ c=c.parentElement; if(!c) break; const inp=c.querySelector('input'); if(inp) return inp; }
      return null;
    }, f.label);
    const el = inp.asElement();
    if(!el){ out[f.label]='NOTFOUND'; continue; }
    await el.click({clickCount:3});
    await el.fill(f.value);
    await el.press('Enter');
    out[f.label] = await el.inputValue().catch(()=>'?');
  }
  return out;
}
```

radio(有锁定期 / 不追保)用 `page.getByText('有锁定期',{exact:true}).click()` 即可,选了"有锁定期"会多出"锁定期(月)"字段(默认 3,若产品是 3 不用改)。填完点"立即分析",等 3~5 秒读结果。

**通毓表单的「末次观察敲出价」字段**选降敲后默认自动联动成 85(首次 − 步长 × 间隔),但本产品它 = 期末障碍价 = 降落伞(60),脚本会填成降落伞值覆盖 85(见上方模板)。填完用 `page.evaluate` 读一遍所有 input.value 确认(尤其末次观察敲出价应为 60 而非 85),再点分析。

## 跑分析 & 读取

1. 点"立即分析"按钮。
2. 等 3~5 秒结果加载。
3. 读取结果区(标题"假设...买入一份该SNOWBALL合约"那块):
   - **胜率**(主要数字,如 98.14%)
   - 平均敲出时间(天)
   - 已完结合约共计(样本数,如 2361)
   - 盈利 / 亏损 笔数与概率(盈利概率 = 胜率)
4. 文案里引用胜率时,带上回测区间和样本数更有可信度,如"2016-07-09 至 2026-07-08 回测区间内…胜率 97.88%"。回测区间用脚本 `date_range` 实际输出,不要套旧默认 2016-06-26~2026-06-25。
5. **胜率截图**(推介材料"产品胜率数据"那一节的图):只截"胜率 + 已完结合约表"统计区——含胜率数字、平均敲出时间、已完结合约共计表(盈利/亏损行)。**不要含**"假设…买入一份该SNOWBALL合约"那行说明文字,也**不要含**上方的"敲入敲出价分布图"。定位:结果区里从"胜率"标签到"亏损"行的连续块;若 DOM 里和"假设…买入"标题共用容器,用 `browser_run_code_unsafe` 截"胜率"文字所在容器到表格的父级(排除标题),或 fullPage 截后用 PIL 裁掉标题行和分布图。**别用 heading 父容器 `xpath=..` 偷懒**——会把"假设…买入"标题也截进去,且可能漏表格。

**胜率定义**:盈利概率 = 敲出(拿票息了结)+ 未敲入未敲出(持有到期拿票息);亏损 = 敲入未敲出。

## 字段会变怎么办

通毓终端是 Vue 单页应用,表单字段随结构选择动态增减、ref 每次交互后可能变。本文档给的是字段**语义映射**(哪个产品参数对应哪个表单字段),不是死的 ref 编号。执行时每次点击/填值后重新快照,按字段名(如"首次观察敲出价""期末障碍价")定位当前 ref,不要记旧 ref。这样 UI 改版也能跟上。
# 胜率截图 bounding-box 硬规则

“产品胜率数据”截图必须先定位结果统计内容容器,再只截该元素 bounding box。不要按浏览器视口截图,不要 full page。
